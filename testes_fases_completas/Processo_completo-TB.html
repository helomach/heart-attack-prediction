Fase 2: Diviso dos Dados

Este arquivo realiza a diviso dos dados, preparando-os para as fases subsequentes de modelagem e treinamento dos algoritmos de aprendizado de mquina. Abaixo est um resumo das principais etapas e funcionalidades implementadas nesta fase.

Etapas da Diviso dos Dados

1. Importao das Bibliotecas

As bibliotecas necessrias para a criao e avaliao dos modelos so importadas, como:

  • pandas para manipulao de dados
  • sklearn.model_selection para a funo train_test_split, que realiza a diviso dos dados

2. Definio das Variveis Independentes e Dependentes

As variveis so definidas para a diviso:

  • X: Contm as variveis independentes, ou seja, todas as colunas exceto a varivel alvo.
  • y: Contm a varivel dependente, que neste caso a coluna que indica se houve um ataque cardaco HadHeartAttack.

3. Diviso dos Dados

Os dados so divididos em conjuntos de treino e teste para avaliar o desempenho dos modelos. Esta diviso realizada utilizando a funo train_test_split do sklearn.

  • Treino (X_train, y_train): Usado para treinar os modelos.
  • Teste (X_test, y_test): Usado para avaliar a performance do modelo.

Fase 3 - Balanceamento dos Dados

Este arquivo implementa o balanceamento das classes da varivel-alvo para evitar que o modelo de aprendizado de mquina se torne enviesado para a classe majoritria. Para isso, foi utilizado o SMOTE (Synthetic Minority Over-sampling Technique), uma tcnica que cria amostras sintticas da classe minoritria, equilibrando assim a distribuio das classes no conjunto de dados.

Etapas do Balanceamento dos Dados

1. Carregamento dos Dados

O conjunto de dados de treino carregado a partir do arquivo salvo na fase anterior, possibilitando a continuao do fluxo de preparao.

2. Aplicao do SMOTE para Balanceamento das Classes

Foi utilizado o SMOTE para gerar amostras sintticas da classe minoritria no conjunto de treino. Essa tcnica ajuda a balancear as classes, evitando que o modelo aprenda de forma enviesada para a classe mais frequente

3. Verificao das Propores das Classes

Aps a aplicao do SMOTE, foi exibido as novas propores das classes no conjunto de dados de treino para confirmar o balanceamento

4. Salvando Dados Balanceados

Por fim, os dados balanceados so salvos em um arquivo para serem utilizados na prxima fase de treinamento e teste dos modelos.
